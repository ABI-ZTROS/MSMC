## Task 1: 修复侧边栏图标裁剪

### 问题分析

折叠态 48px 下图标被裁剪 ~45%：
- `ModernNavListBoxStyle` Margin=8 → 偏移 8px
- `ModernNavListBoxItemStyle` Margin=6, Padding=14 → 偏移 8+6+14=28px
- ListBoxItem 内 `StackPanel` Margin=8 → 偏移 36px
- 图标 22px → 右边缘 36+22=**58px**，超出 48px 可见区 10px

### 修复方案：折叠宽度 48→56px + 重新计算间距

**目标**：56px 折叠宽度下，图标完全可见且大致居中

| 层级 | 原值 | 新值 | 累计偏移 |
|------|------|------|----------|
| ListBox Margin (ModernNavListBoxStyle) | 8 | 4 | 4px |
| ListBoxItem Margin (ModernNavListBoxItemStyle) | 6 | 2 | 6px |
| ListBoxItem Padding | 14 | 8 | 14px |
| StackPanel Margin (ListBoxItem内) | 8 | 4 | 18px |
| 图标宽度 | 22 | 22 | 18+22=**40px** |
| 可见区域 | 48px | **56px** | 右侧余量 16px |

展开态 (240px) 验证：文字起始 18+22+12=52px，可用文字宽 188px → 充足

### 修改文件

1. **`Themes/AppResources.xaml`**
   - `ModernNavListBoxStyle` (~L358): `Margin="8,8"` → `Margin="4,4"`
   - `ModernNavListBoxItemStyle` (~L219): `Margin="6,3"` → `Margin="2,3"`，`Padding="14,10"` → `Padding="8,8"`

2. **`Views/MainWindow.xaml`**
   - NavSidebar `Width="48"` → `Width="56"`
   - 5 个 ListBoxItem 内 StackPanel `Margin="8,6"` → `Margin="4,4"`
   - Storyboard 中 `To="48"` → `To="56"` (CollapseSidebarStoryboard L156)
   - NavHeaderText/NavFooter 内的 Padding/Margin 验证：头部 `Padding="12,16,12,12"` + 图标 32px = 44px ✓; 底部 `Padding="12,10"` + 内部 `Padding="8,6"` + 图标 14px = 34px ✓

3. **`Views/MainWindow.xaml.cs`**
   - `CollapseSidebar()` 中 `DoubleAnimation(48, ...)` → `DoubleAnimation(56, ...)`
   - `NavSidebar.Width = 48` → `NavSidebar.Width = 56`

## Task 3: 嵌入 Space Grotesk 字体 + 移除字体选择器

### 3a. 下载并嵌入字体

1. 下载 Space Grotesk TTF 到 `src/McServerGuard/Resources/Fonts/`
   - 从 Google Fonts GitHub 下载变量字体 `SpaceGrotesk[wght].ttf`
   - 若 WPF .NET 10 不支持变量字体，用 fonttools instancer 生成静态 Regular/SemiBold/Bold

2. **`McServerGuard.csproj`** 添加：
   ```xml
   <ItemGroup>
     <Resource Include="Resources\Fonts\SpaceGrotesk.ttf" />
   </ItemGroup>
   ```

3. **`Themes/AppResources.xaml`** (~L133):
   ```xml
   <FontFamily x:Key="AppFontFamily">pack://application:,,,/McServerGuard;component/Resources/Fonts/#Space Grotesk</FontFamily>
   <FontFamily x:Key="AppMonospaceFontFamily">Consolas, Courier New</FontFamily>
   ```

### 3b. 移除字体选择器，硬编码字体

4. **`Services/ThemeService.cs`**：
   - `IThemeService` 接口：删除 `string FontFamily { get; set; }`
   - `ThemeService`：删除 `_fontFamily` 字段和 `FontFamily` 属性
   - `ThemeSettings`：删除 `FontFamily` 属性（或保留用于反序列化兼容，但不用）
   - `UpdateResources()` 中字体部分：直接用 `new FontFamily("pack://application:,,,/McServerGuard;component/Resources/Fonts/#Space Grotesk")`
   - `ResetToDefault()`：删除 `_fontFamily = "Segoe UI"` 赋值
   - `LoadSettings()`/`SaveSettings()`：不再读写 FontFamily

5. **`ViewModels/SettingsViewModel.cs`**：
   - 删除 `FontFamily` 属性 (~L58-59)
   - 删除 `SystemFonts` 属性 (~L61-77)
   - 删除 `OnFontFamilyChanged` partial 方法（如存在）

6. **`Views/SettingsPage.xaml`**：
   - 删除整个"字体设置"Border (~L437-474)
   - 字体预览块一并删除

### 3c. 替换所有硬编码字体引用

7. **`Themes/AppResources.xaml`** (~L564): `FontFamily="Segoe UI"` → `FontFamily="{DynamicResource AppFontFamily}"`

8. **`Views/MainWindow.xaml`** (~L21): `FontFamily="{DynamicResource MaterialDesignFont}"` → `FontFamily="{DynamicResource AppFontFamily}"`

9. **`Views/UserAgreementWindow.xaml`** (~L11): 同上

10. **所有 `FontFamily="Consolas"` / `FontFamily="Consolas, Courier New"` 替换**为 `{DynamicResource AppMonospaceFontFamily}`：
    - `Views/ServerDetectionPage.xaml` (L187, L542, L680, L738)

## Task 5: 替换硬编码颜色为可主题化资源

### 5a. 在 ThemeService.UpdateResources() 中添加缺失的派生笔刷

**`Services/ThemeService.cs`** `UpdateResources()` 方法中添加：

```csharp
// 信号灯色族（从固定色改为可从 PrimaryColor/AccentColor 派生或独立配置）
// 保持默认值但加入资源系统，至少能被 DynamicResource 引用
resources["GaugeGreenBrush"] = new SolidColorBrush(Color.FromRgb(0x4C, 0xAF, 0x50));
resources["GaugeYellowBrush"] = new SolidColorBrush(Color.FromRgb(0xFF, 0xC1, 0x07));
resources["GaugeRedBrush"] = new SolidColorBrush(Color.FromRgb(0xF4, 0x36, 0x4C));

// 危险色（用于删除/停止按钮）
resources["DangerBrush"] = new SolidColorBrush(Color.FromRgb(0xE5, 0x39, 0x35));

// ErrorText 关联 AccentColor
resources["ErrorTextBrush"] = new SolidColorBrush(_accentColor);

// 主色半透明（用于标题栏渐变等）
resources["PrimarySubtleBackgroundBrush"] = new SolidColorBrush(
    Color.FromArgb(0x1A, _primaryColor.R, _primaryColor.G, _primaryColor.B));
```

### 5b. 替换 Views 中的硬编码颜色

| 文件 | 行号 | 原值 | 替换为 |
|------|------|------|--------|
| `ServerDetectionPage.xaml` | ~L300 | `#4CAF50` | `{DynamicResource GaugeGreenBrush}` |
| `ServerDetectionPage.xaml` | ~L470 | `#E53935` | `{DynamicResource DangerBrush}` |
| `SettingsPage.xaml` | ~L634,639,665,670 | `#FB7185` | `{DynamicResource AccentTextBrush}` |
| `MainWindow.xaml` | ~L174 | `#1A3B82F6` | `{DynamicResource PrimarySubtleBackgroundBrush}` |
| `Controls/ColorPickerControl.xaml` | ~L23 | `#FF3B82F6` | `{DynamicResource NavItemSelectedBrush}` |
| `Controls/ColorPickerControl.xaml` | ~L71 | `#FF3B82F6` | `{DynamicResource NavItemSelectedBrush}` |

### 5c. AppResources.xaml 中添加新资源键默认值

在现有笔刷区域添加：
```xml
<SolidColorBrush x:Key="DangerBrush" Color="#E53935" />

```

### 不改动的部分（有意硬编码）
- `UserAgreementWindow.xaml` — 血红色主题是设计意图
- `SettingsPage.xaml` 色板按钮的 `Background` 和 `CommandParameter` — 这些是预设色值本身
- `AppResources.xaml` 阴影色 `#000000` — 阴影永远是黑色
- `WindowCloseButtonStyle` 的 `#E81123`/`#F1707A` — Windows 标准关闭红

1. **Space Grotesk 变量字体**：优先尝试变量字体 TTF。若 WPF 不支持，用 fonttools 生成静态权重。字体名在 TTF 内部为 "Space Grotesk"

3. **ThemeSettings.json 兼容**：保留 FontFamily 字段不删（旧配置文件反序列化不报错），但不再使用

6. **颜色预设不覆盖 UserAgreementWindow**：血红色是刻意设计
