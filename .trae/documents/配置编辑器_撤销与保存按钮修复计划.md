# 配置编辑器 · 撤销与保存按钮修复计划

## 一、问题诊断结论

经全面探索，用户反馈的"撤销和保存按钮不存在"实际包含三个独立问题：

### 问题 1：保存/重置按钮被永久灰显（核心 Bug）
- **现象**：按钮定义在 XAML 中且始终 Visible，但因 `IsEnabled=False` 触发样式 `Opacity=0.4`，视觉上像"不存在"
- **根因**：`ConfigEditorViewModel.OnConfigEntryChanged`（Line 790-799）在用户编辑值时**没有设置 `entry.IsModified = true`**
- **后果链**：`IsModified` 永远 false → `HasUnsavedChanges` 永远 false → `CanSaveConfig()`/`CanResetChanges()` 永远返回 false → 按钮永久禁用灰显

### 问题 2：撤销按钮从未实现
- 全项目无 `UndoCommand`、无撤销按钮 XAML 定义
- `ServerConfigEntry.OriginalValue` 属性已存在（Line 47），数据基础具备

### 问题 3：保存时文件被占用无告警
- `ConfigManager.SaveConfigAsync`（Line 132）用 `File.WriteAllTextAsync` 直接写入
- 文件被服务器进程/编辑器占用时抛 `IOException`，当前 ViewModel catch 后仅写日志（Line 402-406），用户无感知

### 改动 1：修复 IsModified Bug（根因修复）

**文件**：`ConfigEditorViewModel.cs`，方法 `OnConfigEntryChanged`（Line 790-799）

**现状**：
```csharp
private void OnConfigEntryChanged(object? sender, System.ComponentModel.PropertyChangedEventArgs e)
{
    if (sender is not ServerConfigEntry entry || e.PropertyName != nameof(ServerConfigEntry.Value))
        return;

    entry.IsValid = entry.Descriptor is null ||
                    _configManager.ValidateValue(entry.Key, entry.SourceFile, entry.Value);

    HasUnsavedChanges = ConfigEntries.Any(ce => ce.IsModified);
}
```

**改为**：在验证值之后、更新 HasUnsavedChanges 之前，补一行 IsModified 赋值：
```csharp
private void OnConfigEntryChanged(object? sender, System.ComponentModel.PropertyChangedEventArgs e)
{
    if (sender is not ServerConfigEntry entry || e.PropertyName != nameof(ServerConfigEntry.Value))
        return;

    // 非撤销操作时才压入撤销栈
    if (!_isUndoing)
    {
        _undoStack.Push((entry, entry.OriginalValue));  // 见改动2
    }

    entry.IsModified = !string.Equals(entry.Value, entry.OriginalValue, StringComparison.Ordinal);

    entry.IsValid = entry.Descriptor is null ||
                    _configManager.ValidateValue(entry.Key, entry.SourceFile, entry.Value);

    HasUnsavedChanges = ConfigEntries.Any(ce => ce.IsModified);
    UndoCommand.NotifyCanExecuteChanged();  // 见改动2
}
```

**原理**：`ServerConfigEntry` 已有 `OriginalValue` 属性（加载时赋值，Line 719），对比当前 `Value` 与 `OriginalValue` 即可判断是否被修改。

### 改动 2：新增逐步撤销功能

**文件**：`ConfigEditorViewModel.cs`

**新增字段**（类顶部私有字段区，约 Line 96 附近）：
```csharp
/// <summary>编辑历史栈 —— 记录每次值变更前的条目引用与原始值，支持逐步撤销</summary>
private readonly Stack<(ServerConfigEntry Entry, string PreviousValue)> _undoStack = new();

/// <summary>撤销操作进行中标志 —— 防止撤销恢复值时再次触发压栈</summary>
private bool _isUndoing;
```

**新增命令**（放在 ResetChanges 命令之后，约 Line 443 附近）：
```csharp
/// <summary>
/// 撤销最近一次配置编辑命令
/// </summary>
/// <remarks>
/// 从编辑历史栈弹出最近一次变更，将该条目的值恢复为变更前的值。
/// 支持连续多次撤销，直至栈空。
/// </remarks>
[RelayCommand(CanExecute = nameof(CanUndo))]
private void Undo()
{
    if (_undoStack.Count == 0) return;

    _isUndoing = true;
    try
    {
        var (entry, previousValue) = _undoStack.Pop();
        entry.Value = previousValue;
        entry.IsModified = !string.Equals(entry.Value, entry.OriginalValue, StringComparison.Ordinal);
    }
    finally
    {
        _isUndoing = false;
    }

    HasUnsavedChanges = ConfigEntries.Any(ce => ce.IsModified);
    UndoCommand.NotifyCanExecuteChanged();
}

/// <summary>
/// 判断是否可执行撤销命令
/// </summary>
/// <returns>若编辑历史栈非空则返回 true</returns>
private bool CanUndo() => _undoStack.Count > 0;
```

**清空撤销栈的时机**（在以下方法中追加 `_undoStack.Clear()`）：
- `SaveConfigAsync` 保存成功后（Line 398 附近，HasUnsavedChanges = false 之后）
- `ResetChanges` 重置后（Line 436 附近）
- `LoadConfigAsync` 加载新文件时（Line 737 附近，HasUnsavedChanges = false 之后）
- `OnServerChanged` 切换服务器时（Line 494 附近）

**修改 `SaveConfigAsync`**（Line 375-407），保存前预检文件占用：
```csharp
[RelayCommand(CanExecute = nameof(CanSaveConfig))]
private async Task SaveConfigAsync()
{
    if (Server is null || string.IsNullOrEmpty(_currentFilePath))
    {
        Log.Debug("🔄 SaveConfig 跳过: Server 为空或路径为空");
        return;
    }

    Log.Information("💾 开始保存配置到 {Path}", _currentFilePath);

    // ★ 新增：保存前检查文件是否被占用
    try
    {
        using var fs = new FileStream(
            _currentFilePath,
            FileMode.Open,
            FileAccess.ReadWrite,
            FileShare.Read);  // 只允许别人读，不允许别人写 —— 测试独占写权限
        // 能打开说明文件未被独占占用，可以继续保存
    }
    catch (IOException ioEx)
    {
        IsSaveError = true;
        SaveStatusMessage = $"文件被占用，保存失败：{ioEx.Message}（请关闭正在使用该文件的程序，如服务器进程或文本编辑器）";
        Log.Warning(ioEx, "⚠️ 配置文件被占用: {Path}", _currentFilePath);
        return;
    }
    catch (UnauthorizedAccessException authEx)
    {
        IsSaveError = true;
        SaveStatusMessage = $"权限不足，保存失败：{authEx.Message}";
        Log.Warning(authEx, "⚠️ 配置文件无写入权限: {Path}", _currentFilePath);
        return;
    }

    try
    {
        var currentConfig = ConfigEntries
            .ToDictionary(entry => entry.Key, entry => entry.Value);

        await _configManager.SaveConfigAsync(_currentFilePath, currentConfig);

        _originalConfig = new Dictionary<string, string>(currentConfig);
        foreach (var entry in ConfigEntries)
        {
            entry.IsModified = false;
        }
        HasUnsavedChanges = false;
        _undoStack.Clear();  // ★ 新增：清空撤销栈

        // ★ 新增：保存成功提示
        IsSaveError = false;
        SaveStatusMessage = $"配置已保存，共 {currentConfig.Count} 项";

        Log.Information("✅ 配置保存成功，共保存 {Count} 项配置", currentConfig.Count);
    }
    catch (IOException ex)
    {
        IsSaveError = true;
        SaveStatusMessage = $"保存失败：{ex.Message}（文件可能被其他程序占用）";
        Log.Error(ex, "💥 配置保存失败（IO异常）: {Message}", ex.Message);
    }
    catch (Exception ex)
    {
        IsSaveError = true;
        SaveStatusMessage = $"保存失败：{ex.Message}";
        Log.Error(ex, "💥 配置保存失败: {Message}", ex.Message);
    }
}
```

同时修改 `ResetChanges`（Line 422-437）追加清空撤销栈和状态消息：
```csharp
[RelayCommand(CanExecute = nameof(CanResetChanges))]
private void ResetChanges()
{
    Log.Information("🔄 重置所有配置变更");

    foreach (var entry in ConfigEntries)
    {
        if (_originalConfig.TryGetValue(entry.Key, out var originalValue))
        {
            entry.Value = originalValue;
            entry.IsModified = false;
        }
    }

    HasUnsavedChanges = false;
    _undoStack.Clear();        // ★ 新增
    UndoCommand.NotifyCanExecuteChanged();  // ★ 新增
    SaveStatusMessage = null;  // ★ 新增：清除状态消息
}
```

### 改动 5：XAML 新增保存状态告警 UI

**文件**：`ConfigEditorPage.xaml`，按钮栏 Border 内部（Line 330-418）

<TextBlock Grid.Row="1" Grid.ColumnSpan="2"
           Text="{Binding SaveStatusMessage}"
           FontSize="12"
           Margin="0,8,0,0"
           Visibility="{Binding SaveStatusMessage, Converter={StaticResource NullToVisibilityConverter}}">
    <TextBlock.Style>
        <Style TargetType="TextBlock">
            <Style.Triggers>
                <DataTrigger Binding="{Binding IsSaveError}" Value="True">
                    <Setter Property="Foreground" Value="#FF5252" />

- DataTrigger：IsSaveError=true 时红色 `#FF5252`，false 时绿色 `#4CAF50`
