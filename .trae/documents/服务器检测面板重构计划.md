# 服务器检测面板重构计划

## 一、用户需求总结

用户对当前 `ServerDetectionPage.xaml` 的布局不满（左右分栏 340+* 显得很尴尬），要求重新设计，并明确指出：

1. **布局风格**：左侧固定侧边栏 + 右侧 Tab 切换
2. **操作入口**：每张卡片有独立操作按钮 + 顶部统一总控栏（都要）
3. **视觉重心**：突出主要操作（启动/导入） / 服务器列表紧凑+状态色 / 启动命令预览显眼
4. **特别提醒**：注意功能重合和冲突，写好**功能锁**防止出错

## 二、当前问题分析

### 2.1 布局层面（XAML）
- [ServerDetectionPage.xaml:92-96](file:///workspace/src/McServerGuard/Views/ServerDetectionPage.xaml#L92-L96) 固定 340+* 双栏，左侧要塞下 4 个 Card（操作栏、运行中、已知、日志），右栏要塞下 5 个 Card（操作、详情、JVM 参数、保存按钮），导致：
  - 左侧 4 个 Card 堆叠，滚动条频繁出现
  - 右侧 JVM 参数 Card 高度 600+，永远要滚
  - 启动/停止按钮在右栏顶部，与"运行中/已知"列表分离，操作路径太长
- 没有状态色：运行中/已停止/未知 全是同一种图标
- 启动命令预览藏在 JVM 参数 Card 最底部，完全不显眼

### 2.2 功能冲突层面（ViewModel）
[ServerDetectionViewModel.cs](file:///workspace/src/McServerGuard/ViewModels/ServerDetectionViewModel.cs) 中已经存在但不够完善的功能锁问题：

| 场景 | 现状 | 风险 |
|------|------|------|
| `IsStarting`（启动当前服务器）和 `IsStartingKnown`（启动已知服务器） | 两个独立 bool | 同时启动两个不同服务器互不阻塞，可能导致 JVM 内存爆掉 |
| `IsStopping` 没有对应的 Known 版本 | — | 已知服务器没停止命令 |
| `IsDetecting` 和 `IsStarting` | `CanDetect = !IsDetecting` 单独判断 | 检测时仍可启动服务器 |
| 编辑 JVM 参数时启动服务器 | 无锁 | 用户编辑一半点启动，使用半成品参数 |
| `SaveCurrentToKnown` 在 `StartCurrentServerAsync` 中自动调用 | 自动保存 | 与底部"保存为已知服务器"按钮功能重复，参数来源不明确 |
| `BrowseAndImportServer` 也会启动服务器 | 一键导入+启动 | 与"启动服务器"按钮冲突，状态混乱 |

### 2.3 缺失能力
- 状态色：没有"运行中绿色 / 停止灰色 / 启动中黄色"区分
- 搜索/过滤：服务器列表多了之后无法快速定位
- 选中高亮：没有清晰的"当前选中"状态
- 启动命令预览：藏得太深

## 四、详细改动

### 4.1 ViewModel 改造 — [ServerDetectionViewModel.cs](file:///workspace/src/McServerGuard/ViewModels/ServerDetectionViewModel.cs)

**新增/重命名字段：**
```csharp
// 统一功能锁（防止任何操作冲突）
[ObservableProperty] private bool _isBusy;          // 总开关：进行任何操作时为 true
[ObservableProperty] private string _busyReason = string.Empty;  // 显示当前锁的原因
[ObservableProperty] private ServerOperation _activeOperation = ServerOperation.None;

public enum ServerOperation
{
    None, Detecting, Importing, Starting, Stopping, SavingConfig, Deleting
}
```

**重写 CanExecute 规则：**
- `CanStartCurrent = !IsBusy && 当前服务器未运行`
- `CanStopCurrent = !IsBusy && 当前服务器在运行`
- `CanDetect = !IsBusy`
- `CanImport = !IsBusy`
- `CanStartKnown = !IsBusy && 目标服务器未运行`
- `CanRemoveKnown = !IsBusy && 目标服务器未运行`（删除正在运行的服务器 = 灾难）

**新增辅助属性：**
```csharp
// 当前服务器运行状态（用于状态色 + 总控按钮文案）
public ServerStatus CurrentServerStatus { get; } // 枚举：Running/Stopped/Starting/Stopping/Unknown
public Brush CurrentServerStatusBrush { get; }    // 对应颜色
public string CurrentServerStatusText { get; }    // "🟢 运行中 PID:1234" 等

// 过滤后的服务器列表（搜索）
public ICollectionView FilteredRunningServers { get; }
public ICollectionView FilteredKnownServers { get; }
[ObservableProperty] private string _searchKeyword = string.Empty;
```

**修改现有方法：**
- `StartCurrentServerAsync` / `StopCurrentServerAsync` / `StartKnownServerAsync`：开头 `if (IsBusy) return;`，中间用 `try/finally` 锁 `IsBusy = true`
- `SaveCurrentToKnown`：从 `StartCurrentServerAsync` 中拆出来，**不再自动保存**（避免与"保存到已知"按钮冲突）
- `BrowseAndImportServer`：拆成两步——`BrowseServer`（仅导入到列表） + 显式 `ImportAndStart`
- `SaveAsKnownServerCommand`：`IsBusy` 锁，避免编辑 JVM 参数到一半被触发

**删除/合并重复功能：**
- 删掉 `StartCurrentServerAsync` 中的 `SaveCurrentToKnown(jvmArgs)` 自动调用
- 删掉 `BrowseAndImportServer` 中的自动启动逻辑

## 七、风险评估

| 风险 | 等级 | 缓解 |
|------|------|------|
| Converter 命名冲突 | 低 | 用 `ServerStatusToXxxConverter` 前缀 |
| TabControl 默认样式与项目主题不一致 | 中 | 用 MaterialDesign 的 TabControl 样式 |
| 移除 `SaveCurrentToKnown` 自动调用破坏现有用户预期 | 中 | 在控制台 Tab 显式提供"保存到已知"按钮，并在按钮上加提示文案 |
| 功能锁过严导致操作不能并发 | 中 | 锁粒度按"服务器级"而非"全局级"——同一时间只锁一个服务器，允许不同服务器的不同操作（视实现复杂度） |
| 列表 hover 按钮在触摸设备无法触发 | 低 | 触摸设备长按显示按钮（暂不实现，记录为后续 TODO） |
