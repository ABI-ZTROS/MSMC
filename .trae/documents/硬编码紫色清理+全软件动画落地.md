# 硬编码紫色清理 + 全软件动画落地 实施计划

> **用户诉求:**
> 1. "那些被硬编码为紫色的东西到底是什么情况" —— 怀疑仍有紫色硬编码没跟随主题
> 2. "我要的全软件动画为什么一点都没有" —— 动画设置（EnableAnimations/AnimationDuration）没生效，且很多页面根本没有动画

---

## 探勘结论

### 问题 1：硬编码紫色 — 只剩 2 处数据级硬编码（可接受）

**全项目扫描结果：**

| [SettingsPage.xaml L251](file:///workspace/src/McServerGuard/Views/SettingsPage.xaml#L251) | `#7B1FA2` | "赛博紫"预设方案的预览小方块 | ✅ **数据级硬编码** — 预览图就是要显示方案的代表色，合理 |

**结论：紫色硬编码已基本清零。** 前几轮 Task 5/6 已经把页面级硬编码替换为 DynamicResource，ThemeService.UpdateResources() 也已覆盖主色/强调色/背景色/卡片色/文字色/边框色共 6 族。

**但是**，MainWindow 的 5 处 `Foreground="White"` 是命名颜色硬编码，不跟随新的 `TextColor` 设置 — 用户改文字色为绿色，标题栏字还是白色。这是真实缺口。

### 问题 2：动画 — 有"半成品"动画，完全没接入 ThemeService

**动画现状盘点：**

| 页面 | 有动画吗？ | 动画内容 | 时长 | 接 ThemeService？ |
|------|-----------|---------|------|------------------|
| **ServerDetectionPage** | ✅ 有 | 卡片入场（Opacity 0→1 + TranslateTransform.X -20→0）+ 空状态淡入 + Loading图标旋转 | 硬编码 0.4s / 0.5s / 2s | ❌ 不读 EnableAnimations，不读 AnimationDuration |
| **ConfigEditorPage** | ✅ 有 | 分组卡片入场（Opacity 0→1 + Y -20→0）+ Expander展开淡入 + Loading旋转 | 硬编码 0.3s / 0.25s / 2s | ❌ 同上 |
| **SystemMonitorPage** | ❌ 无 | — | — | — |
| **AIGuardPage** | ❌ 无 | — | — | — |
| **ServerImportPage** | ❌ 无 | — | — | — |
| **SettingsPage** | ❌ 无 | — | — | — |
| **页面切换（MainWindow ContentControl）** | ❌ 无 | 注释写"六大页面切换（带动画）"但**实际无任何 Storyboard**，Opacity="0" 挂在那没人动 | — | — |
| **导航选中态** | ❌ 无过渡 | 直接切颜色，无 ColorAnimation | — | — |

**ThemeService 的动画属性：** `AnimationDuration`(默认300ms) 和 `EnableAnimations`(默认true) —— **App 里没有任何地方读这俩值**。设置页滑块拖了白拖。

**核心问题：**
1. **4 个页面 + 页面切换 + 导航选中** 没有入场/过渡动画
2. 已有动画的 2 个页面**硬编码时长**，不跟随设置
3. `EnableAnimations` 开关完全没接

## 设计思路

### 颜色部分（小修）
- MainWindow 5 处 `White` → `{DynamicResource MaterialDesignBody}` 或 `{DynamicResource SwatchHoverBorderBrush}`（跟随文字色）
- 注：SettingsPage 色板按钮/预设预览的紫色是数据本身，保留

### 动画部分（大戏）—— 分层实现

**Layer 1: 全局动画资源**（AppResources.xaml）
- 定义全局 `EntryStoryboard`（Opacity 0→1 + 轻微位移）、`FadeInStoryboard`、页面切换 `PageTransitionStoryboard`
- 时长绑定 ThemeService.AnimationDuration？不行，Storyboard Duration 是 Freezable 不能绑。用另一种方式：**在 code-behind 里动态构建 Storyboard** 或用 `VisualStateManager`
- 更简洁的方案：**在 ThemeService 暴露 `EntryAnimationDuration` 属性，各页面 UserControl 在 Loaded 事件里从 ThemeService 读取时长，动态创建并启动 DoubleAnimation**

**Layer 2: 页面切换动画**（MainWindow）
- ContentControl 切换页面时，先淡出旧内容，再淡入新内容
- 用 MainWindow.xaml.cs 监听 CurrentPage 变化，触发 Storyboard
- 或者用 MaterialDesign 的 `Transitioner` 控件（如果 MaterialDesign 有）

**Layer 3: 4 个空白页加入场动画**
- SystemMonitorPage / AIGuardPage / ServerImportPage / SettingsPage 各加 Loaded 触发的淡入动画
- 样式跟 ServerDetectionPage 一致：Opacity 0→1 + 轻微位移

**Layer 4: 已有 2 个页面接入 ThemeService**
- ServerDetectionPage / ConfigEditorPage 的硬编码时长改为从 ThemeService 读取
- EnableAnimations=false 时跳过动画

**最简实现方案（避免过度设计）：**
- 不搞全局 Storyboard 资源（因为 Duration 不能绑定）
- 统一用 **code-behind + 动态创建 DoubleAnimation** 方式
- 写一个 `AnimationHelper` 静态类封装 `FadeIn(UIElement element, double durationMs)` / `SlideInFromBottom(...)` / `SlideInFromLeft(...)`
- 各页面在 `OnLoaded` 里调用 `AnimationHelper.FadeAndSlideIn(this, _themeService.AnimationDuration)`
- EnableAnimations=false 时直接返回不播

## Task 2: 新建 AnimationHelper 静态类

**Files:** `/workspace/src/McServerGuard/Views/Helpers/AnimationHelper.cs`（新建）

**How:**
```csharp
namespace McServerGuard.Views.Helpers;

using System.Windows;
using System.Windows.Media.Animation;

public static class AnimationHelper
{
    // 淡入 + 从下方滑入（页面入场）
    public static void FadeAndSlideIn(UIElement element, int durationMs, double slideDistance = 20)
    {
        if (durationMs <= 0) return;

        element.Opacity = 0;
        var translate = new TranslateTransform(0, slideDistance);
        element.RenderTransform = translate;

        var opacityAnim = new DoubleAnimation(1, TimeSpan.FromMilliseconds(durationMs))
        {
            EasingFunction = new CubicEase { EasingMode = EasingMode.EaseOut }
        };
        var yAnim = new DoubleAnimation(0, TimeSpan.FromMilliseconds(durationMs))
        {
            EasingFunction = new CubicEase { EasingMode = EasingMode.EaseOut }
        };

        element.BeginAnimation(UIElement.OpacityProperty, opacityAnim);
        translate.BeginAnimation(TranslateTransform.YProperty, yAnim);
    }

    // 仅淡入
    public static void FadeIn(UIElement element, int durationMs)
    {
        if (durationMs <= 0) return;
        element.Opacity = 0;
        var anim = new DoubleAnimation(1, TimeSpan.FromMilliseconds(durationMs))
        {
            EasingFunction = new CubicEase { EasingMode = EasingMode.EaseOut }
        };
        element.BeginAnimation(UIElement.OpacityProperty, anim);
    }

    // 页面切换：淡出当前 + 淡入新内容
    public static void CrossFade(UIElement oldElement, UIElement newElement, int durationMs)
    {
        // 淡出旧的（如果有）
        if (oldElement != null && durationMs > 0)
        {
            var fadeOut = new DoubleAnimation(0, TimeSpan.FromMilliseconds(durationMs / 2));
            oldElement.BeginAnimation(UIElement.OpacityProperty, fadeOut);
        }
        // 淡入新的
        if (newElement != null && durationMs > 0)
        {
            newElement.Opacity = 0;
            var fadeIn = new DoubleAnimation(1, TimeSpan.FromMilliseconds(durationMs / 2))
            {
                BeginTime = TimeSpan.FromMilliseconds(durationMs / 2)
            };
            newElement.BeginAnimation(UIElement.OpacityProperty, fadeIn);
        }
    }
}
```

**Why:** 统一动画工具，避免每个页面重复写 Storyboard XAML。时长从 ThemeService 传入，天然跟随设置。

## Assumptions & Decisions

1. **SettingsPage 色板/预设的紫色硬编码保留** — 这些是"数据"（色板的显示颜色、预设方案的代表色），不是"主题"。改主题色不会让"紫色按钮"变成别的颜色，符合预期。
2. **用 code-behind + AnimationHelper 而非纯 XAML Storyboard** — 因为 Storyboard 的 Duration 是 Freezable 不能绑定，无法跟随 ThemeService.AnimationDuration。code-behind 动态构建动画是 WPF 里的标准做法。
3. **页面切换用 CrossFade（淡入淡出）** — 简单优雅，性能好。不搞左右滑动（需要测量内容宽度，容易踩坑）。
4. **EnableAnimations=false 时 durationMs=0** — AnimationHelper 里判断 <=0 直接返回，等于跳过动画。统一入口，不用每个页面都写 if。
5. **导航选中过渡可选** — 优先级低于入场动画和页面切换，看时间是否允许。

## Verification Steps

1. 编译无 CS#### 错误
2. 启动后切换页面有淡入淡出效果
3. 改 AnimationDuration 滑块 → 下次切页/下次加载页面时动画时长变化
4. 关 EnableAnimations → 切页/入场都无动画，瞬间显示
5. 改文字色 → 标题栏/状态栏/导航选中文字也变
6. ServerDetectionPage/ConfigEditorPage 入场动画时长也跟随设置变化
