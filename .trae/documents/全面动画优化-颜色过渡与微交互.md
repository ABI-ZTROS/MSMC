| **Button 悬停/按下** | MaterialDesign 自带（有波纹效果） | ✅ 已有，不用管 |
| **Snackbar 弹出** | MaterialDesign 自带（有滑入） | ✅ 已有，不用管 |
| **Slider 拖动** | MaterialDesign 自带（有 thumb 动画） | ✅ 已有，不用管 |

### 核心难点：WPF 颜色过渡动画
WPF 的 `Trigger` + `Setter` 是**瞬间赋值**，没有过渡。要做颜色过渡动画，有两条路：

1. **VisualStateManager + ColorAnimation** — 标准做法，在 ControlTemplate 里定义 VisualStateGroup，每个状态用 Storyboard 做 ColorAnimation。改动大但最正规。
2. **AnimatedBrush 附加属性** — 写一个自定义 Behavior/AttachedProperty，监听属性变化，用 DoubleAnimation 驱动 Brush.Color 插值。通用但复杂度高。
3. **方案折中** — 对关键控件（导航项、色板、卡片）用 VisualStateManager 加颜色过渡；对全局主题切换这种"全 App 颜色跳变"暂不处理（技术上可行但要改几十个控件模板，ROI 低）。

**本计划选方案 3**：聚焦最显眼、最常交互的控件，用 VisualStateManager 加颜色过渡。全局主题色渐变暂不做（改主题这种低频操作，瞬间跳变可接受）。

## Task 1: 导航项颜色过渡动画（最高优先级）

**Files:** `/workspace/src/McServerGuard/Themes/AppResources.xaml`

**What:** `NavListBoxItemStyle` 的 ControlTemplate 从 `Trigger` + `Setter` 改为 `VisualStateManager` + `ColorAnimation`，让：
- 正常 → 悬停：背景色 `CardBackgroundBrush → NavItemHoverBrush` 平滑过渡
- 正常 → 选中：背景色 `Transparent → NavItemSelectedBrush` + 左侧竖线 `Transparent → NavItemSelectedIndicatorBrush` 平滑过渡
- 选中 → 选中+悬停：背景色 `NavItemSelectedBrush → NavItemSelectedHoverBrush` 平滑过渡
- 全部 200ms CubicEase Out

**How:**
1. 保留 `ControlTemplate` 结构（Border + SelectionIndicator + ContentPresenter）不变
2. 把 `<ControlTemplate.Triggers>` 整个删掉
3. 在 Template 的根 Border 里加 `<VisualStateManager.VisualStateGroups>`
4. 定义 3 个 VisualStateGroup：
   - `CommonStates`（Normal / MouseOver）
   - `SelectionStates`（Unselected / Selected）
   - 组合状态用 `VisualStateManager` 自动处理
5. 每个 VisualState 里用 `Storyboard` + `ColorAnimation` 驱动 `Border.Background.Color` 和 `SelectionIndicator.Background.Color`

**注意:** WPF 的 VisualState 是**状态机**，CommonStates 和 SelectionStates 是两个独立的 group，可以同时生效。悬停+选中的叠加由 WPF 自动混合。

**时长:** 跟随 ThemeService？不行，VisualState 的 Storyboard Duration 是 Freezable 不能绑。**硬编码 200ms**——微交互动画 200ms 是行业标准，用户一般不会去调这个。如果后续要接 ThemeService，可以用 code-behind 动态改，但 ROI 低。

## Task 2: 色板悬停过渡动画

**Files:** `/workspace/src/McServerGuard/Themes/AppResources.xaml`

**What:** `ColorSwatchStyle` 的 ControlTemplate 从 Trigger 改为 VisualStateManager，悬停时边框色 + 阴影透明度平滑过渡（200ms），按下缩放也加动画（150ms）。

**How:**
1. 保留 ControlTemplate 结构
2. `<ControlTemplate.Triggers>` → `<VisualStateManager.VisualStateGroups>`
3. CommonStates:
   - Normal: BorderBrush=Transparent, DropShadowEffect.Opacity=0
   - MouseOver: BorderBrush=SwatchHoverBorderBrush, DropShadowEffect.Opacity=0.5
   - Pressed: Scale 0.9
4. 用 ColorAnimation + DoubleAnimation，全部 150-200ms

## Task 3: 卡片悬停过渡动画（EnhancedCardStyle）

**Files:** `/workspace/src/McServerGuard/Themes/AppResources.xaml`

**What:** `EnhancedCardStyle` 悬停背景色从瞬切改为 200ms 渐变。同时加一个轻微的上浮效果（TranslateTransform Y -2px）增强质感。

**How:** 因为 `EnhancedCardStyle` 是 `Style` 不是 `ControlTemplate`，不能直接用 VisualStateManager。有两个选择：
- A: 改模板，加一层 Border 包内容，在里面写 VisualStateManager
- B: 用 `EventTrigger` + `EnterActions/ExitActions` + `ColorAnimation`（Style.Triggers 支持）

**选方案 B**，改动最小：
```xml
<Style.Triggers>
    <Trigger Property="IsMouseOver" Value="True">
        <Trigger.EnterActions>
            <BeginStoryboard>
                <Storyboard>
                    <ColorAnimation To="..." Duration="0:0:0.2" />
                    <DoubleAnimation To="-2" Duration="0:0:0.2" />
                </Storyboard>
            </BeginStoryboard>
        </Trigger.EnterActions>
        <Trigger.ExitActions>
            <BeginStoryboard>
                <Storyboard>
                    <ColorAnimation To="..." Duration="0:0:0.2" />
                    <DoubleAnimation To="0" Duration="0:0:0.2" />
                </Storyboard>
            </BeginStoryboard>
        </Trigger.ExitActions>
    </Trigger>
</Style.Triggers>
```
同时要给 Card 加 `RenderTransform`（TranslateTransform）作为 Setter。

## Task 4: ServerDetection / ConfigEditor 卡片动画接入 ThemeService

**Files:**
- `/workspace/src/McServerGuard/Views/ServerDetectionPage.xaml`
- `/workspace/src/McServerGuard/Views/ConfigEditorPage.xaml`

**What:** 把这两个页面里卡片级入场动画的硬编码时长改为可配置。但卡片在 DataTemplate/Style 里，Storyboard Duration 不能直接绑定。

**方案 A（推荐）: 保留硬编码 250ms** — 卡片入场动画很快（0.25s），用户感知不到时长差异，不值得大动干戈。

**方案 B（激进）: 用 Attached Property 动态注入 Duration** — 写一个 `AnimationHelper.Duration` 附加属性，在 PropertyChangedCallback 里遍历 VisualTree 找 Storyboard 改 Duration。复杂度高，ROI 低。

**选方案 A**，在计划里明确说明"卡片级微动画保持 250ms 硬编码，因为过快/过慢都会破坏节奏感"。

## Task 5: 主题色切换全局过渡（可选，高级）

**What:** 用户改主题色（主色/强调色/背景色等）时，全 App 颜色平滑过渡而非瞬间跳变。

**技术方案:** 利用 WPF 的 `AnimatedObservableObject` 模式或写一个 `ThemeAnimationService`，在 `ApplyTheme()` 之前先存旧值，然后对所有关键 brush 做 ColorAnimation 插值到新值。

**复杂度:** 极高。需要对 `Application.Current.Resources` 里的每个 SolidColorBrush 做动画，而且要处理 Brush 是 frozen 的问题。ROI 很低（改主题是低频操作，用户一天改不了几次）。

**建议: 不做**。把精力放在 Task 1-3（日常交互的微动画）上。如果用户后续强烈要求，再加。

## Assumptions & Decisions

1. **微交互动画时长统一 200ms 硬编码** — 200ms 是 Material Design 标准微动画时长，用户几乎不会去调这个。接 ThemeService 的话需要大量 code-behind/附加属性，ROI 极低。
2. **用 VisualStateManager 还是 Trigger.EnterActions?** — ControlTemplate 级别的（导航项、色板）用 VisualStateManager（标准做法）；Style 级别的（EnhancedCardStyle）用 Trigger.EnterActions/ExitActions（改动最小）。
3. **卡片入场动画保留硬编码 250ms** — 节奏感比可配置性更重要。
4. **不做全局主题色渐变** — 低频操作 + 极高复杂度 + 容易出 bug。
5. **MaterialDesign 自带动画不动** — Toggle/Button/Snackbar/Slider 已有动画，重复定义会冲突。
