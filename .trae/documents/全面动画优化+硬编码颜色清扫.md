## Part B: 动画优化方案

### 现状盘点

**已有动画（上轮完成）**
- 主窗口启动淡入 + 6 个页面入场动画（FadeAndSlideIn，跟随 ThemeService）
- ServerDetection / ConfigEditor 卡片级入场（硬编码 0.25s）
- ConfigEditor Expander 展开淡入（硬编码 0.2s）
- ColorSwatch 按下缩放（硬编码 Scale 0.9）

**MaterialDesign 自带动画（不用管）**
- Toggle 开关滑入、Button 波纹、Snackbar 弹出、Slider 拖动

**无动画 / 瞬间切换（本轮要加）**

| 区域 | 现状 | 加什么动画 |
|------|------|-----------|
| 导航项悬停 | Trigger 瞬切背景色 | 200ms 背景色渐变 |
| 导航项选中 | Trigger 瞬切背景 + 左侧竖线 | 200ms 背景色 + 竖线颜色渐变 |
| 卡片悬停（EnhancedCardStyle） | Trigger 瞬切背景 | 200ms 背景色渐变 + 轻微上浮（Y -2px）|
| 色板悬停 | Trigger 瞬切边框 + 阴影 | 200ms 边框色 + 阴影透明度渐变 |
| 色板按下 | Trigger 瞬切 Scale | 150ms 缩放动画 + 回弹 |

**不做的（说明理由）**
- **全局主题色渐变过渡**：改主题是低频操作，全 App 颜色渐变技术复杂度极高（要对几十个 Brush 做 ColorAnimation，还要处理 Frozen 问题），ROI 极低
- **卡片入场动画接 ThemeService**：卡片级微动画 250ms 是黄金时长，节奏感比可配置性重要，且 Storyboard Duration 是 Freezable 不能直接绑

## Task 1: 清扫 9 处硬编码颜色

### 1.1 ServerDetectionPage 遮罩色
- **文件:** `Views/ServerDetectionPage.xaml`
- **改动:** `Background="#CC000000"` → `Background="{DynamicResource LoadingOverlayBrush}"`
- **验证:** ThemeService 已经在 UpdateResources() 里更新这个资源（第 177 行），用的是 cardBg 的 0xCC 透明度版本

### 1.2 SettingsPage 6 个 HintAssist.Hint 颜色
- **文件:** `Views/SettingsPage.xaml` + `Views/SettingsPage.xaml.cs` + `ViewModels/SettingsViewModel.cs`
- **问题:** HintAssist.Hint 是 `object?` 类型，可以传 Color 也可以传 Brush。但不能直接绑字符串 Hex（`PrimaryColorHex` 是字符串）。
- **方案 A（推荐）:** 在 SettingsViewModel 里加 6 个 `*ColorBrush` 属性（已有 `PrimaryColorBrush` 等？需要确认），直接绑 Brush
- **方案 B:** 用 `Binding + Converter` 把 Hex 字符串转 Color
- **选方案 A**，和现有的 `PrimaryColorBrush` 模式一致。检查一下是否已有 BackgroundColorBrush / CardColorBrush 等

### 1.3 SystemMonitorPage Cyan 图标
- **文件:** `Views/SystemMonitorPage.xaml:132`
- **改动:** `Foreground="Cyan"` → `Foreground="{DynamicResource AccentTextBrush}"`
- **理由:** 内存图标用强调色，和下面内存折线图用 AccentTextBrush 保持一致

### 1.4 TrendChartControl 默认 LineColor
- **文件:** `Views/Controls/TrendChartControl.cs`
- **改动:** 把 `LineColorProperty` 的默认值从硬编码紫色改为 `null` 或 `Brushes.Transparent`，然后在 `OnLoaded` 时用 `TryFindResource("PrimaryHueMidBrush")` 作为回退
- **模式:** 和 GaugeRingControl 的 `GetTrackBrush()` / `GetLabelBrush()` 完全一致
- **注意:** 因为 LineColor 是依赖属性，用户在 XAML 里显式设置了（SystemMonitorPage 里设置了 `LineColor="{DynamicResource GaugeGreenBrush}"` 和 `LineColor="{DynamicResource AccentTextBrush}"`），所以默认值只在"用户没传"时生效，不影响现有使用

## Task 2: 导航项颜色过渡动画（最高优先级）

- **文件:** `Themes/AppResources.xaml`
- **目标:** `NavListBoxItemStyle` 的 ControlTemplate 从 `Trigger` + `Setter` 改为 `VisualStateManager` + `ColorAnimation`
- **效果:**
  - 正常 → 悬停：背景色 200ms 渐变（CardBg → NavHover）
  - 正常 → 选中：背景色 + 左侧竖线 200ms 渐变
  - 选中 + 悬停叠加：自动混合
- **实现方式:**
  - 保留 ControlTemplate 结构（Border + SelectionIndicator + ContentPresenter）
  - 删除 `<ControlTemplate.Triggers>`
  - 在根 Border 内加 `<VisualStateManager.VisualStateGroups>`
  - 定义两个 VisualStateGroup：`CommonStates`（Normal/MouseOver）和 `SelectionStates`（Unselected/Selected）
  - 每个 VisualState 用 `Storyboard` + `ColorAnimation` 驱动 `Border.Background.Color` 和 `SelectionIndicator.Background.Color`
- **时长:** 200ms 硬编码（Material Design 标准微动画时长，用户几乎不会调）
- **Easing:** `CubicEase EasingMode.EaseOut`

## Assumptions & Decisions

1. **微交互动画时长统一 200ms 硬编码** — Material Design 标准，ROI 低
2. **ControlTemplate 级用 VisualStateManager，Style 级用 Trigger.EnterActions** — 改动最小原则
3. **预设色板/快速方案预览保留硬编码** — 它们是数据，不是界面样式
4. **Gauge 绿/黄/红保留硬编码** — 语义色（好/中/差），不随主题变
5. **不做全局主题色渐变** — 低频操作 + 极高复杂度
6. **卡片入场动画保留 250ms 硬编码** — 节奏感 > 可配置性
7. **TrendChartControl 网格/标签灰色暂不改** — 目前只有暗色主题，白色半透网格合理；等以后有亮色主题再说
