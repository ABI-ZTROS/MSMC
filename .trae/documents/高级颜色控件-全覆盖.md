# 高级颜色控件 — 让自定义颜色真正覆盖整个软件 实施计划

> **背景:** 用户反馈"设置里面只有两个调节UI不应该是更多吗"。核查确认：当前只有主色+强调色 2 个颜色控件，且 `SubtleBorderBrush`/`DeepBackgroundBrush`/`CardSubtleBorderBrush`/窗口背景 `MaterialDesignPaper`/正文 `MaterialDesignBody` 都是固定值，改主色时不变色——这是"看不到效果"的根因。

**Goal:** 新增 4 个高级颜色控件（背景色/卡片色/文字色/边框色），每个独立可调，通过 `ThemeService.UpdateResources()` 覆盖对应 brush（含 MaterialDesign 内置键），让用户改任一颜色都能立即看到全 App 变化。总颜色控件数从 2 → 6。

---

## 设计

新增 4 个主题属性，每个映射到一组 brush：

| 新控件 | ThemeService 属性 | 覆盖的 brush | 默认值 |
|--------|-------------------|-------------|--------|
| 背景色 | `BackgroundColor` | `MaterialDesignPaper`（窗口/页面底色）、`DeepBackgroundBrush`（=暗化 0.3） | `#0F0F1A` |
| 卡片色 | `CardColor` | `CardBackgroundBrush`、`CardHoverBrush`（=亮化 0.08）、`NavItemHoverBrush`（=亮化 0.15）、`TerminalBackgroundBrush`（=暗化 0.1）、`LoadingOverlayBrush`（=半透明卡片色） | `#1E1E2E` |
| 文字色 | `TextColor` | `MaterialDesignBody`、`MaterialDesignBodyLight`（=alpha 180） | `#E0E0E0` |
| 边框色 | `BorderColor` | `SubtleBorderBrush`、`CardSubtleBorderBrush`（=alpha 0x33） | `#2A2A4A` |

**保留不动:** 主色→`NavItemSelectedBrush`/`AccentGradientBrush`/`PrimarySubtleBorderBrush`/`NavItemSelectedIndicatorBrush`/`NavItemSelectedHoverBrush`/MaterialDesign `PrimaryHue*`；强调色→`AccentTextBrush`/`AccentSubtleBorderBrush`/MaterialDesign `SecondaryHue*`；语义色（仪表盘绿黄红/成功信息错误色）固定。`SwatchHoverBorderBrush` 保持白色固定（色板悬停高亮）。

## Task 1: IThemeService + ThemeService 新增 4 个颜色属性

**Files:** `/workspace/src/McServerGuard/Services/ThemeService.cs`

**How:**
1. `IThemeService` 接口加 4 个属性：`Color BackgroundColor { get; set; }`、`CardColor`、`TextColor`、`BorderColor`。
2. `ThemeService` 加 4 个私有字段 + 公共属性（模式同 `PrimaryColor`：set 时 `ApplyTheme()`）：
   - `_backgroundColor = Color.FromRgb(0x0F, 0x0F, 0x1A)`
   - `_cardColor = Color.FromRgb(0x1E, 0x1E, 0x2E)`
   - `_textColor = Color.FromRgb(0xE0, 0xE0, 0xE0)`
   - `_borderColor = Color.FromRgb(0x2A, 0x2A, 0x4A)`
3. `UpdateResources()` 改造（替换原 `cardBg = DarkenColor(_primaryColor, 0.85)` 逻辑）：
   ```csharp
   var cardBg = _cardColor;
   var cardHover = LightenColor(cardBg, 0.08);
   var navHover = LightenColor(cardBg, 0.15);
   var terminalBg = DarkenColor(cardBg, 0.1);

   resources["CardBackgroundBrush"] = new SolidColorBrush(cardBg);
   resources["CardHoverBrush"] = new SolidColorBrush(cardHover);
   resources["NavItemHoverBrush"] = new SolidColorBrush(navHover);
   resources["TerminalBackgroundBrush"] = new SolidColorBrush(terminalBg);
   resources["LoadingOverlayBrush"] = new SolidColorBrush(Color.FromArgb(0xCC, cardBg.R, cardBg.G, cardBg.B));

   // 背景色：覆盖 MaterialDesign 窗口底色 + 深背景
   resources["MaterialDesignPaper"] = new SolidColorBrush(_backgroundColor);
   resources["DeepBackgroundBrush"] = new SolidColorBrush(DarkenColor(_backgroundColor, 0.3));

   // 文字色：覆盖 MaterialDesign 正文 + 次要文字
   resources["MaterialDesignBody"] = new SolidColorBrush(_textColor);
   resources["MaterialDesignBodyLight"] = new SolidColorBrush(Color.FromArgb(180, _textColor.R, _textColor.G, _textColor.B));

   // 边框色
   resources["SubtleBorderBrush"] = new SolidColorBrush(_borderColor);
   resources["CardSubtleBorderBrush"] = new SolidColorBrush(Color.FromArgb(0x33, _borderColor.R, _borderColor.G, _borderColor.B));
   ```
   保留主色/强调色派生的其他键不变。
4. `LoadSettings()` 读取 4 个新字段；`SaveSettings()` 写入；`ResetToDefault()` 重置 4 个新字段为默认值。

## Assumptions & Decisions

1. **CardBackgroundBrush 改为独立 CardColor 控制**（不再从 PrimaryColor 派生）：用户要"更多控件"，独立控制比派生更符合需求。改主色不再改卡片色，但用户可单独调卡片色。默认值 #1E1E2E 与原 AppResources 一致，视觉无回归。
2. **覆盖 MaterialDesignPaper / MaterialDesignBody**：这两个是 MaterialDesign 内置键，`UpdateResources()` 在 `PaletteHelper.SetTheme()` 之后运行，覆盖生效。DynamicResource 引用会拿到新值。这会让所有 MaterialDesign 控件的底色/文字色跟随用户设置——正是"覆盖整个软件"。
3. **MaterialDesignBodyLight 用 alpha 180 派生**：保持"次要文字比正文暗"的语义，跟随 TextColor。
4. **SwatchHoverBorderBrush 保持白色固定**：色板悬停高亮用白色最显眼，不随边框色变。
5. **预设方案扩展为 6 色**：让"快速方案"真正一键换肤，体现"更多控件"的价值。
6. **不引入 per-color 预设色板**：4 个新颜色各给 HEX 输入 + 预览即可，避免 UI 过载；预设方案提供快速换肤。

## Verification Steps

1. 编译无 CS#### 错误。
2. 启动后设置页有 6 个颜色控件（主色/强调色/背景色/卡片色/文字色/边框色）。
3. 改背景色 → 窗口底色立即变。
4. 改卡片色 → 所有卡片背景变。
5. 改文字色 → 所有正文变。
6. 改边框色 → 所有边框变。
7. 点"赛博紫"等预设 → 6 色协同变化。
8. 重启 App → 颜色持久化。
