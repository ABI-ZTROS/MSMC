# 服务器检测淡入淡出 + 主题色全覆盖 实施计划

> **For agentic workers:** 必读评估文档 `/workspace/.trae/documents/丝滑UI与动画方案评估.md`，动画统一用 `RenderTransform`/`Opacity`，禁用 `Margin/Width`。

**Goal:**
1. 给服务器检测页加一个检测过程的淡入淡出遮罩 + 结果入场动画，把"识别和分析"包装成一个有视觉时长的过程（掩盖 WMI 卡顿、提供仪式感）
2. 把所有散落在 XAML/C# 中的硬编码颜色改为引用主题资源，让设置里的自定义颜色真正覆盖到整个软件

**Architecture:**
- 检测页：在 ScrollViewer 内套一层 Grid，叠加一个 `IsDetecting` 绑定的半透明遮罩 Border（参考 [ConfigEditorPage.xaml](file:///workspace/src/McServerGuard/Views/ConfigEditorPage.xaml#L509-L544) 的 loading 遮罩范式）；结果卡片用 `Loaded` 事件触发 `Opacity` + `TranslateTransform.X` 淡入滑动动画（参考 [ConfigEditorPage.xaml](file:///workspace/src/McServerGuard/Views/ConfigEditorPage.xaml#L30-L44)）
- 主题色：扩展 [ThemeService.UpdateResources()](file:///workspace/src/McServerGuard/Services/ThemeService.cs#L111-L131) 让原本"假装可主题化"的 `CardBackgroundBrush`/`CardHoverBrush`/`NavItemHoverBrush` 也从主色派生；补齐 [AppResources.xaml](file:///workspace/src/McServerGuard/Themes/AppResources.xaml) 缺失的语义资源键；把所有 XAML 硬编码颜色改为 `DynamicResource`；把 [GaugeRingControl.cs](file:///workspace/src/McServerGuard/Views/Controls/GaugeRingControl.cs) / [TrendChartControl.cs](file:///workspace/src/McServerGuard/Views/Controls/TrendChartControl.cs) 的静态 Brush 改为 `TryFindResource` 动态读取

**Tech Stack:** WPF + MaterialDesignInXamlToolkit + CommunityToolkit.Mvvm

## Task 1: 给 ServerDetectionPage 加检测遮罩 + 结果入场动画

**Files:**
- Modify: `/workspace/src/McServerGuard/Views/ServerDetectionPage.xaml`

**What/Why:** 当前检测期间页面无任何 loading 视觉反馈，结果"啪"一下硬切出现，体验生硬。加一个 `IsDetecting` 绑定的半透明遮罩（旋转 Loading 图标 + "正在识别和分析..."文字），检测完成后遮罩淡出、结果卡片以淡入+滑动入场。这既是视觉缓冲（掩盖 WMI 卡顿），又把"识别和分析"包装成有仪式感的过程。

**How:**

1. 在 `<UserControl.Resources>` 中定义一个入场动画 Storyboard 样式（参考 [ConfigEditorPage.xaml](file:///workspace/src/McServerGuard/Views/ConfigEditorPage.xaml#L13-L45) 的 `ConfigItemCardStyle`），但适配服务器卡片：
   - `CubicEase` 缓动资源
   - `ServerCardStyle`（TargetType=Border）：`RenderTransform` = `TranslateTransform X=-20`，`EventTrigger Loaded` 触发 `DoubleAnimation` Opacity 0→1 (250ms) + TranslateTransform.X -20→0 (300ms)

2. 把 `<ScrollViewer>` 内容用一个 `<Grid>` 包起来，在 Grid 内叠加一个检测遮罩 Border：
   ```xml
   <Border Background="#CC000000"
           CornerRadius="12"
           Visibility="{Binding IsDetecting, Converter={StaticResource BoolToVisibilityConverter}}">
       <StackPanel HorizontalAlignment="Center" VerticalAlignment="Center">
           <materialDesign:PackIcon Kind="Loading" Width="48" Height="48"
                                    HorizontalAlignment="Center" Margin="0,0,0,12">
               <materialDesign:PackIcon.RenderTransform>
                   <RotateTransform x:Name="LoadingIconRotate" CenterX="24" CenterY="24" />
               </materialDesign:PackIcon.RenderTransform>
           </materialDesign:PackIcon>
           <materialDesign:PackIcon.Triggers>
               <EventTrigger RoutedEvent="Loaded">
                   <BeginStoryboard>
                       <Storyboard>
                           <DoubleAnimation Storyboard.TargetName="LoadingIconRotate"
                                            Storyboard.TargetProperty="Angle"
                                            From="0" To="360" Duration="0:0:1"
                                            RepeatBehavior="Forever" />
                       </Storyboard>
                   </BeginStoryboard>
               </EventTrigger>
           </materialDesign:PackIcon.Triggers>
           <TextBlock Text="正在识别和分析服务器..." FontSize="14"
                      HorizontalAlignment="Center" Opacity="0.8" />
       </StackPanel>
   </Border>
   ```
   - 遮罩颜色 `#CC000000` 是半透明黑（不依赖主色，合理硬编码，作为通用遮罩）

3. 把检测结果 `ItemsControl` 的 `ItemTemplate` 里那个 `<materialDesign:Card>` 外层套一个 `<Border Style="{StaticResource ServerCardStyle}">`，让每张结果卡片入场时淡入+从左滑入

4. 把空状态 `StackPanel`（第 76-88 行）的 `Visibility` 绑定保持不变，但额外加 `Opacity` 初始值 0 + `Loaded` 触发 `DoubleAnimation` Opacity 0→1 (200ms)，避免遮罩淡出时空状态硬切

5. 给主窗口"开始检测"按钮加 `IsEnabled` 绑定（防止重复点击），见 Task 2

## Task 4: 修复 ThemeService.UpdateResources() 让"假装可主题化"的笔刷真正从主色派生

**Files:**
- Modify: `/workspace/src/McServerGuard/Services/ThemeService.cs`（`UpdateResources()` 第 111-131 行）

**What/Why:** 当前 `CardBackgroundBrush`/`CardHoverBrush`/`NavItemHoverBrush` 被 `UpdateResources()` 覆盖却用写死的 `Color.FromRgb(30,30,46)` 等值，给人"已主题化"假象，实则不随主色变。用户要求"自定义颜色覆盖到整个软件"，这三个必须改。

**How:**

1. `CardBackgroundBrush`：从主色派生——取主色的暗化版（`DarkenColor(_primaryColor, 0.85)`），让卡片背景带有主色色调但很暗。新增 `DarkenColor` 工具方法（与 `LightenColor` 对称）
2. `CardHoverBrush`：比 `CardBackgroundBrush` 稍亮（`LightenColor(cardBg, 0.08)`）
3. `NavItemHoverBrush`：比 `CardHoverBrush` 再稍亮（`LightenColor(cardBg, 0.15)`）
4. `TerminalBackgroundBrush`：也改为从主色派生的极暗色（`DarkenColor(_primaryColor, 0.92)`），让终端背景也带主色色调
5. `AccentTextBrush`、`NavItemSelectedBrush`、`AccentGradientBrush` 保持现状（已正确从主色/强调色派生）

**注意:** 仪表盘绿/黄/红是语义色，**不**从主色派生，保持固定。`SubtleBorderBrush` 等暗色键也保持固定（暗色主题下合理）。

## Task 7: GaugeRingControl.cs / TrendChartControl.cs 静态 Brush 改为动态读取主题资源

**Files:**
- Modify: `/workspace/src/McServerGuard/Views/Controls/GaugeRingControl.cs`
- Modify: `/workspace/src/McServerGuard/Views/Controls/TrendChartControl.cs`

**What/Why:** 这两个自绘控件当前用 `static readonly Brush` 在类加载时固化颜色，无法响应主题切换。用户改主色后，仪表盘和趋势图的背景/默认线色不会变。改为 `TryFindResource` 动态读取。

**How:**

**GaugeRingControl.cs:**
- 保留 `GreenBrush`/`YellowBrush`/`RedBrush` 及对应 Pen 为静态（语义色，不随主题变）
- `TrackBrush`/`LabelBrush`/`WhiteBrush` 改为在 `OnRender` 中通过 `TryFindResource("CardBackgroundBrush")` 等动态读取，找不到则回退到静态默认值
- 实现 `OnLoaded` 时调用 `InvalidateVisual()` 触发重绘（确保主题切换后重绘）

**TrendChartControl.cs:**
- `GridPenBrush`/`AxisLabelBrush`/`NoDataBrush`/`HaloBrush` 保留静态（这些是中性色，暗色主题下合理）
- `LineColorProperty` 的默认值从硬编码 `#7B1FA2` 改为在 `OnLoaded` 时从 `TryFindResource("PrimaryHueMidBrush")` 读取并 `SetValue`，让趋势线默认跟随主色
- 实现 `OnLoaded` 触发 `InvalidateVisual()`

**注意:** `TryFindResource` 必须在控件已加入逻辑树后调用（`OnLoaded` 时机安全），构造函数里调用会返回 null。

## Assumptions & Decisions

1. **"淡入淡出效果来为软件提供识别和分析翻译时间"** 解读为：检测期间显示遮罩（掩盖 WMI 卡顿）+ 检测完成后结果卡片淡入入场。不改动 ServerDetector 内部逻辑，只做视觉层。
2. **"自定义颜色覆盖到整个软件"** 解读为：所有 XAML 硬编码颜色改为 `DynamicResource`，`ThemeService.UpdateResources()` 让所有派生笔刷从主色计算，自绘控件动态读取主题资源。语义色（绿/黄/红）保持固定不随主色变。
3. **遮罩颜色 `#CC000000`** 保持硬编码：这是通用半透明黑遮罩，不依赖主题，合理。
4. **SettingsPage 预设色板按钮的 HEX 值** 保留硬编码：这是调色板数据本身，不是样式。
5. **`AppResources.xaml` 中的颜色值** 保留硬编码：这是主题字典的单一来源，是其职责。
6. **不引入新 NuGet 包**：所有改动用 WPF 原生 `DynamicResource` + `TryFindResource`。
7. **动画时长** 统一用 250-300ms + `CubicEase EaseOut`，符合 [丝滑UI与动画方案评估.md](file:///workspace/.trae/documents/丝滑UI与动画方案评估.md) 的规范。
