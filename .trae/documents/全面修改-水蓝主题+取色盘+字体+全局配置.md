## Task 2: 新增取色盘（Color Picker）

### 2.1 技术选型

WPF 没有内置 ColorPicker。三个选项：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A: WinForms ColorDialog 互操作** | 零依赖，系统原生 | 丑陋，与 WPF 风格不搭 |
| **B: 手写简易取色器** | 无依赖，完全可控 | 工作量大，功能简陋 |
| **C: 用 MaterialDesign 的 ColorZone + 自定义滑块组合** | 风格统一，不增依赖 | 需自己组装，复杂度中等 |
| **D: 引入 NuGet 包 (WPFColorPicker 或 Extended.Wpf.Toolkit)** | 功能完整 | 新增依赖，可能有许可问题 |

**推荐方案 B：手写简易取色器**
- 三个 Slider（R/G/B 或 H/S/V）+ 预览框
- 旁边保留 HEX 输入框
- 放在颜色预览 Border 旁边，点击预览打开 Popup 形式的取色面板
- 风格与 MaterialDesign 统一

原因：项目是工具软件，取色器不需要太专业，简单 RGB 滑块 + HEX 输入足够。零新增依赖。

### 2.2 实现方式

**新增控件：`Views/Controls/ColorPickerControl.xaml` + `ColorPickerControl.xaml.cs`**
- 一个 UserControl
- 内部：RGB 三个 Slider + HEX TextBox + 实时预览方块
- 依赖属性：`SelectedColor`（Color 类型，双向绑定）
- 显示方式：Popup，点击预览色板弹出

**SettingsPage.xaml 改动：**
- 每个颜色设置（主色/强调色/背景色/卡片色/文字色/边框色）的预览 Border 改为可点击，点击弹出取色器
- 保留现有 HEX 输入框 + 色板预设
- 取色器作为增强，不替代现有功能

### 2.3 改动文件
- 新建 `Views/Controls/ColorPickerControl.xaml` + `.cs`
- 修改 `SettingsPage.xaml` — 6 个颜色预览处加入取色器触发
- 修改 `SettingsPage.xaml.cs` — 处理取色器交互

## Task 4: 全局配置文件 + 已知服务器持久化

### 4.1 设计目标

- 检测到的服务器可以"保存"到已知服务器列表
- 下次打开软件自动加载已知服务器
- 服务器可以命名、备注、分组
- 已知服务器信息持久化到 JSON 文件

### 4.2 数据模型

**新建 `Models/KnownServer.cs`：**
```csharp
public class KnownServer
{
    public string Id { get; set; } = Guid.NewGuid().ToString();
    public string Name { get; set; } = string.Empty;          // 用户自定义名称
    public string ServerJarPath { get; set; } = string.Empty;  // JAR 路径（唯一标识）
    public string WorkingDirectory { get; set; } = string.Empty;
    public string JavaPath { get; set; } = string.Empty;
    public long MaxHeapMemoryBytes { get; set; }
    public int Port { get; set; } = 25565;
    public string? Notes { get; set; }                          // 备注
    public string Group { get; set; } = "默认";                 // 分组
    public DateTime AddedAt { get; set; } = DateTime.Now;
    public DateTime LastSeenAt { get; set; } = DateTime.Now;
    public bool IsFavorite { get; set; }
    // 从检测结果匹配：通过 ServerJarPath 对应
}
```

### 4.3 全局配置文件结构

**新建 `Services/AppConfigService.cs`：**
```csharp
public class AppConfig
{
    public int Version { get; set; } = 1;
    public List<KnownServer> KnownServers { get; set; } = [];
    // 未来可扩展：窗口位置、上次打开的页面等
}
```

文件路径：`%AppData%/McServerGuard/app-config.json`

与 `theme-settings.json` 关系：
- `theme-settings.json` — 仅主题设置（继续保留，不迁移）
- `app-config.json` — 应用全局配置（服务器、窗口状态等）

两个独立文件，各司其职。

### 4.4 服务接口

```csharp
public interface IAppConfigService
{
    AppConfig Config { get; }
    void Load();
    void Save();
    void AddKnownServer(KnownServer server);
    void RemoveKnownServer(string id);
    void UpdateKnownServer(KnownServer server);
    KnownServer? FindByJarPath(string jarPath);
}
```

### 4.5 DI 注册

在 `App.xaml.cs` 的 Services 中注册 `IAppConfigService`，单例模式。

## 风险与注意事项

1. **字体大小不统一修改** — 各区域字号不同（标题18/正文13/小字11），统一改 FontSize 会破坏布局。第一版只改 FontFamily。
2. **取色器用 Popup 还是 Dialog？** — 用 Popup 轻量，不阻塞。放在颜色预览框下方弹出。
3. **已知服务器与检测结果的对应关系** — 用 `ServerJarPath` 作为唯一标识。检测到 JAR 路径匹配的服务器时，自动标记为"已保存"。
4. **全局配置与主题配置分离** — 两个 JSON 文件，避免一个文件太大导致损坏影响全部。
5. **水蓝色对比度验证** — 确保文字色与背景色对比度足够（WCAG AA 标准）。
